# Quick Start

agent-html turns semantic `.agent.html` documents into stable, shareable HTML artifacts.

Most users only need this flow:

1. Run `ahtml`
2. Get the writing prompt
3. Write `.agent.html`
4. Render HTML

## 1. Install the CLI [#install-the-cli]

`@agent-html/ahtml` is published once to npmjs.com. Pick your package manager:

`ahtml` starts the guided entry flow. On first run it prepares the managed runtime under `~/.ahtml`. It does not write `src/`, `components.json`, `vite.config.ts`, or `agent-html.project.json` into the current repository.

## 2. Optional: install the ahtml skill [#install-the-ahtml-skill]

If you use the `skills` CLI with Codex or other agents, install the `ahtml` skill:

```bash
npx skills add Sayhi-bzb/Agent-HTML --skill ahtml
```

## 3. Get the writing prompt [#get-the-writing-prompt]

```bash
ahtml prompt
```

Use the schema output as the source of truth for registered components, allowed props, allowed children, and approved document style config choices.

## 4. Write a document [#write-a-document]

A valid document needs a `<page>` root.

```html title="artifact.agent.html"
<meta-agent style-ref="report-default" />

<page title="Review">
  <card title="Summary">
    This review is a stable HTML artifact instead of a long Markdown note.
  </card>
</page>
```

## 5. Build the artifact [#build-the-artifact]

```bash
ahtml build artifact.agent.html
```

`build` validates the document before generating the static artifact directory.

## 6. Preview locally [#preview-locally]

```bash
ahtml preview artifact.agent.html
```

Preview serves the same static output produced by `build`. It does not use a separate renderer.

## 7. Preview a style gallery [#preview-a-style-gallery]

```bash
ahtml gallery
```

Gallery opens the style editor and showcase canvas. Use it to manage local style ids, adjust tokens and treatments, and choose the current default style used by `preview` or `build` when a document does not set `style-ref` explicitly.
