# Getting Started with agent-html

agent-html is a local CLI engine for agents that need to produce safe, inspectable static artifacts.

The `ahtml` package owns the config, engine, CLI, and shadcn-backed managed artifact runtime. Runtime files live under `~/.ahtml` or `%USERPROFILE%\.ahtml` by default, with `AHTML_HOME` as an override. Agents do not write arbitrary HTML, CSS, Tailwind classes, scripts, event handlers, React props, Radix props, or shadcn props.

<Callout title="Primary rule">
  Install `ahtml`, then treat the schema as the source of truth for component
  names, props, children, and render config values. Use `ahtml setup` when you
  want guided runtime setup; runtime-aware commands can still repair defaults
  automatically.
</Callout>

## 1. Install the CLI [#install-the-cli]

`@agent-html/ahtml` is published once to npmjs.com. Pick your package manager:

  <CodeBlockTabsList>
    <CodeBlockTabsTrigger value="npm">npm</CodeBlockTabsTrigger>
    <CodeBlockTabsTrigger value="pnpm">pnpm</CodeBlockTabsTrigger>
    <CodeBlockTabsTrigger value="yarn">yarn</CodeBlockTabsTrigger>
    <CodeBlockTabsTrigger value="bun">bun</CodeBlockTabsTrigger>
  </CodeBlockTabsList>

  <CodeBlockTab value="npm">

```bash
npm install -g @agent-html/ahtml
ahtml setup
ahtml status
ahtml doctor
```

  </CodeBlockTab>

  <CodeBlockTab value="pnpm">

```bash
pnpm add -g @agent-html/ahtml

ahtml status
ahtml doctor
```

  </CodeBlockTab>

  <CodeBlockTab value="yarn">

```bash
yarn global add @agent-html/ahtml
ahtml setup
ahtml status
ahtml doctor
```

  </CodeBlockTab>

  <CodeBlockTab value="bun">

```bash
bun add -g @agent-html/ahtml
ahtml setup
ahtml status
ahtml doctor
```

  </CodeBlockTab>
</CodeBlockTabs>

`ahtml setup` configures the managed runtime under `~/.ahtml`, currently with a shadcn-backed renderer. Guided setup defaults to `--component-source shadcn-cli`, which installs selected runtime components through `npx shadcn@latest` inside the managed runtime. `ahtml status`, `ahtml doctor`, `ahtml build`, and `ahtml preview` initialize or repair the managed runtime automatically with bundled defaults when setup has not run. They do not write `src/`, `components.json`, `vite.config.ts`, or `agent-html.project.json` into the current repository.

`ahtml status` and `ahtml doctor` can show a non-blocking package update hint. Set `AHTML_NO_UPDATE_CHECK=1` to disable that check.

## 2. Read the schema [#read-the-schema]

```bash
ahtml schema --format prompt
```

Use the schema output to learn the registered components, allowed props, allowed children, and finite render config values.

## 3. Write a document [#write-a-document]

A valid document needs a `<page>` root.

```html title="artifact.agent.html"
<meta-agent
  theme="neutral"
  density="comfortable"
  tone="report"
  width="article"
/>

<page title="Review">
  <card title="Summary">
    This artifact was written with standard agent-html components.
  </card>
</page>
```

## 4. Validate before building [#validate-before-building]

```bash
ahtml validate --input artifact.agent.html
```

Validation catches unknown components, unknown props, blocked implementation details, invalid children, and invalid render config values before artifact generation.

## 5. Build the artifact [#build-the-artifact]

```bash
ahtml build --input artifact.agent.html --out dist/html
```

The output is a static artifact directory that can be opened, shared, deployed, or archived.

## 6. Preview locally [#preview-locally]

```bash
ahtml preview --input artifact.agent.html --out dist/html --port 4173
```

Preview serves the same static output produced by `build`. It does not use a separate renderer.

## Inspect the result [#inspect-the-result]

```bash
ahtml inspect --input artifact.agent.html
```

Use `inspect` to review effective config and component usage without inspecting runtime files.
