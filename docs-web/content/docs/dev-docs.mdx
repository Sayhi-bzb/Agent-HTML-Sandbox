# Dev Docs

## Repository development [#repository-development]

```bash
npm install
npm link
ahtml init
ahtml status
ahtml doctor
ahtml schema --format prompt
```

Repository fallback:

```bash
npm run agent -- schema --format prompt
```

Local check levels:

```bash
npm run check:quick
npm run check:ready
npm run check:docs
npm run check:all
```

Use `check:quick` while editing, `check:ready` before merge or release, `check:docs` for documentation changes, and `check:all` for CI-equivalent coverage.

## Verify package boundary [#verify-package-boundary]

```bash
npm pack --dry-run
```

The package should include only the config, engine, CLI, bin entry, TypeScript configs, and README. It should not include a root Vite app, package-local renderer, root shadcn UI files, docs web files, examples, specs, blueprints, or tests.

## Verify installed CLI behavior [#verify-installed-cli-behavior]

```bash
npm run verify:pack
```

This packs the project, installs it into a temporary consumer project, and verifies installed `ahtml` commands.

## Verify published alpha [#verify-published-alpha]

```bash
npm view @agent-html/ahtml dist-tags versions --json
npm install @agent-html/ahtml@alpha
```

The public package is published under `@agent-html/ahtml`; the CLI bin remains `ahtml`.

## Publish npm package [#publish-npm-package]

Publishing is tag-driven through GitHub Actions and npm Trusted Publishing. Do not use a long-lived `NPM_TOKEN` for releases.

Configure the npm trusted publisher for `@agent-html/ahtml`:

```txt
Provider: GitHub Actions
Owner: Sayhi-bzb
Repository: Agent-HTML-Sandbox
Workflow filename: publish-npm.yml
```

Release an alpha:

```bash
npm version 0.1.0-alpha.1 --no-git-tag-version
git add package.json package-lock.json
git commit -m "chore: release 0.1.0-alpha.1"
git tag v0.1.0-alpha.1
git push origin main --tags
```

Prerelease versions publish to the `alpha` dist-tag. Stable versions publish to `latest`. The publish workflow fails if the Git tag does not match `package.json` version.

## Distribute the ahtml skill [#distribute-the-ahtml-skill]

The `ahtml` agent skill lives at `.agents/skills/ahtml`. The open `skills` CLI discovers `.agents/skills/` automatically.

List skills from this repository:

```bash
npx skills add Sayhi-bzb/Agent-HTML-Sandbox --list
```

Install the `ahtml` skill for Codex:

```bash
npx skills add Sayhi-bzb/Agent-HTML-Sandbox --skill ahtml -a codex
```

Install globally for Codex:

```bash
npx skills add Sayhi-bzb/Agent-HTML-Sandbox --skill ahtml -a codex -g
```

Test installation from a local checkout:

```bash
npx skills add . --skill ahtml -a codex --copy -y
```

## Runtime architecture [#runtime-architecture]

```txt
installed ahtml package
  -> config + engine + CLI
  -> user-local Vite + shadcn integration
  -> static artifact directory
```

`ahtml build` must use the user-local integration. Do not restore a package-local Vite builder, package-local renderer, or package-local shadcn UI kit.

```txt
Build command: npm run docs:web:build
Output directory: docs-web/out
```
