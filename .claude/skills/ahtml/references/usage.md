# ahtml Usage

Use this when the user wants to produce, validate, build, preview, or inspect an artifact.

## User Flow

```txt
user asks
  -> agent reads ahtml schema
  -> agent outputs structured composition or .agent.html
  -> ahtml validates and sanitizes
  -> user-local Vite + shadcn renders
  -> static artifact directory
```

## Commands

Inspect the agent-facing contract:

```bash
ahtml schema --format prompt
```

Compose structured JSON into the standard document:

```bash
ahtml compose --input composition.json --out artifact.agent.html
```

Validate without building:

```bash
ahtml validate --input artifact.agent.html
```

Build the static artifact:

```bash
ahtml build --input artifact.agent.html --out dist/html
```

Preview locally:

```bash
ahtml preview --input artifact.agent.html --out dist/html --port 4173
```

Inspect document or built artifact metadata:

```bash
ahtml inspect --input artifact.agent.html
ahtml inspect --dir dist/html
```

Read or set finite render config:

```bash
ahtml config get
ahtml config set density compact
```

## Composition Shape

Use finite metadata and component nodes. Keep implementation details out of the document.

```json
{
  "meta": {
    "theme": "neutral",
    "density": "comfortable",
    "tone": "report",
    "width": "article"
  },
  "document": {
    "name": "page",
    "props": {
      "title": "CLI Demo"
    },
    "children": [
      {
        "name": "card",
        "props": {
          "title": "Overview"
        },
        "children": ["Generated by compose."]
      }
    ]
  }
}
```

Never add Tailwind classes, `className`, `style`, event handlers, scripts, arbitrary HTML attributes, raw HTML, Radix props, or full shadcn props to agent-facing input.
